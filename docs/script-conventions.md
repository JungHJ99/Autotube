# 스크립트 & 메타데이터 컨벤션 — autotube

> `kpop-script-writer` 호출 시 적용할 톤·구조의 정본. **기본 에이전트 정의는 중립 분석가 톤이라,
> 아래 (A)(B)·제목 톤 등은 호출 prompt에 매번 명시 override 해야 한다.**

## 1. 단일 문서 deep-dive (paper-first, not angle-first)

각도를 먼저 정하고 자료를 모으지 않는다. 최신 해외 논문/보고서 **한 편**을 먼저 찾고 영상 한 편
분량으로 샅샅이 분석한다. 보조 자료는 1–2개까지, 중심은 항상 primary source.
구성: 문서 소개 → 방법론 → 핵심 발견들 → 함의 (비디오 에세이).
**Why:** 멀티소스 라운드업은 cherry-pick 같고 깊이가 얕으며 "이 논문 한 편이 이렇게 말한다"는 훅이 죽는다.

## 2. 시의성 체크 (framing freshness)

문서가 최신이어도 프레이밍이 stale일 수 있다. 핵심 인물/사건 상태(군 복무·복귀·은퇴·정책 변경)를
빠르게 확인. **Why:** "BTS 공백기" 프레이밍을 잡았는데 이미 복귀한 상태여서 어그로가 안 끌린 사례(2026-05-12).
데이터는 살리되 프레이밍이 dated면 우회하거나 다른 문서 재선정.

## 3. 톤 — 전면 찬양 + 분석가/저널리스트 (구어체 X)

- **무조건적 찬양 톤.** 비판/균형/약점 지적·"다만 짚어둘 부분" 류 금지. 채널 정체성=외부 자료가
  한국을 어떻게 인정하는지 자랑스럽게 풀기 (7:3 비판 비율은 사용자가 명시 취소 2026-05-12).
- **분석가 톤** (침착맨/슈카 썰풀이체 X). 합쇼체("...입니다", "...로 나타났습니다") 위주, "보고서에 따르면",
  "연구진은 ~로 측정했다" 식 근거 명시. 14F·슈카·비즈니스 인사이드 영상 에세이 느낌.
- 찬양은 직접 자랑("역시 한국")이 아니라 **숫자와 출처가 자랑을 대신**하게. 인용·외부 기관명·수치 최대 동원.
- **짧은 문장 유지(35자 이하). TTS 친화는 양보 X.**

### 긴장감 빌더 자주 (beat 전환 5–7개 중 2–3개)
"여기서 끝이 아닙니다." / "심지어." / "이게 다가 아닙니다." / "더 충격적인 건" /
"그런데 진짜 무게는 여기서부터입니다." 류. 클라이맥스 직전 한 번은 거의 필수. 매번 다른 표현으로.

## 4. male commentary 3-step (호출 시 명시 override 필요)

매 commentary 세그먼트는:
1. 직전 female 번역에서 핵심 문장 짚기 ("방금 매체가 '~'라고 표현했지?")
2. 그게 무슨 뜻인지 풀어 설명 (배경/맥락/구체 수치)
3. 한국에게 어떤 의미인지 확장

찬양적 어조(자긍심 자극)는 유지하되 **호들갑/감탄사("대박!","와!","쩐다!") 금지** — 분석가가 사실로 자긍심을 자극.

### ⚠️ 금지/절제 표현 (2026-06-01 사용자 지적)
- **"이게 진짜 한국이 한 거다"** — 절대 금지(영상마다 박혀 구려졌음).
- **"정말 미친 숫자다"** — 영상당 1회, 진짜 충격 수치에만.
- "다른 어떤 나라도 못 한 일이다" — 진짜 사실일 때만.
- patriot 클리셰 패키지를 **prompt 예시로 박지 말 것** — 에이전트가 복붙해서 매 commentary에 나옴.

**권장 대체:** 수치를 다른 거대 사건과 비교("마이클 잭슨 배드 투어 34만 vs BTS 84만") / 외신 직접 인용
("빌보드가 'top-grossing'이라고 썼다") / 비교 우위 명시 / 결론을 fact로 마무리(감탄으로 끝내지 말 것).

## 5. 세그먼트 구조 (총 11–17개)

```
intro(male) + bridge(male, 고정형식 "오늘은 ~에서 발표된 ~ 한번 보시죠.")
  + (translation(female) + commentary(male)) × 4–7
  + closing(male, 아래 sign-off 필수)
```

## 6. 채널 sign-off (모든 영상 closing 끝, 글자 그대로)

**TTS `text` 필드** (`파이널 케이` — TTS가 영문 K를 어색하게 읽음):
> 저희는 최고의 전문성을 가지고 여러분들께 사실만을 전달드리는 한국 관련 소식 리뷰 유튜버, 파이널 케이입니다. 저희가 리뷰하는 다음 소식도 듣고싶다면, 구독 좋아요 부탁드립니다.

**자막 `subtitle_text` 필드** (`파이널K` — 정식 브랜드 표기):
> 저희는 최고의 전문성을 가지고 여러분들께 사실만을 전달드리는 한국 관련 소식 리뷰 유튜버, 파이널K입니다. 저희가 리뷰하는 다음 소식도 듣고싶다면, 구독 좋아요 부탁드립니다.

closing의 정리+forward look 뒤 자연스러운 호흡만 두고 붙인다(transition 큐 X). 썸네일 우하단 워드마크도 `파이널K`.
**한 번 누락해 재합성한 사례 있음** → script-writer 정의 + subtitle-normalizer 정의 + 이 문서 3중으로 박아둠.
채널명 변경 시 4곳(kpop-script-writer.md, subtitle-normalizer.md, build_thumbnail.py, 이 문서) + MEMORY 인덱스 동시 갱신.

## 7. 제목/썸네일 톤 — 단언적 단문 호들갑 (분석가형 X, override 필요)

기본 에이전트는 분석가형 제목을 만든다. 사용자가 원하는 건 **단언적 단문 호들갑**.

**YouTube 제목:**
- 형식 `[외신 반응] X가 Y했다` / `[외신 반응] X는 더 이상 ~가 아니다`. **부제(— 어쩌고) 금지, 한 문장 단문.**
- 강동사로 단언: 무너졌다/항복했다/점령했다/갈아치웠다/삼켜버렸다. 충격/분석/리포트/심층 같은 분석가 단어 빼고 결과만.
- 예: "[외신 반응] 라스베이거스가 BTS 앞에 무릎을 꿇었다"
- 헤드라인 3마디 구조(`[인용/사건]+[세계 반응]+[reveal 명사]`)와 호들갑 단어는 [thumbnail-conventions.md](thumbnail-conventions.md).

**제목 ≠ 썸네일:** 제목과 썸네일 띠 문구를 같은 메시지로 쓰지 말 것(제목 슬롯 낭비). 썸네일이 A 앵글이면
제목은 B 앵글(다른 수치/기록/선언). 썸네일 핵심 단어를 제목에서 반복 금지.
